ТЗ: Разработка Frontend-части образовательного сервиса
1. Общие требования
Адаптивность: Интерфейс должен корректно отображаться на десктопах и планшетах (основная рабочая среда).

Рендеринг HTML: Задачи приходят из БД с HTML-тегами — фронтенд должен безопасно и красиво их отрисовывать (библиотеки типа html-react-parser).

Математика: Поддержка отображения формул (рекомендуется MathJax или KaTeX).

2. Основные компоненты интерфейса
2.1. Сайдбар (Навигация)
Дерево тем: Список блоков, внутри которых номера заданий.

Индикаторы статуса: * Серая иконка — не приступал.

Желтая — в процессе (есть ошибки).

Зеленая — решено верно.

Переключатель режимов: Toggle-кнопка между «Разбор» и «Практика».

2.2. Рабочая область задания
Заголовок: Номер и название задачи.

Контент: Блок с текстом задачи (HTML), изображениями и ссылками на файлы.

Поле ввода ответа (Динамическое):

Если answer_type == "number": Простой input.

Если answer_type == "pair": Два поля x и y.

Если answer_type == "table": Сетка (Grid) из инпутов, размер которой задается метаданными задачи.

Кнопка «Проверить»: Анимация загрузки и мгновенный фидбек (верно/неверно).

2.3. ИИ-Ассистент (Chat Widget)
Окно чата, которое можно свернуть/развернуть.

Режим «Разбор»: Кнопка «Помоги сделать первый шаг» или «Объясни алгоритм».

Режим «Практика»: Появляется автоматически при неверном ответе с предложением разобрать ошибку.

Ограничение: В режиме «Вариант/Экзамен» этот компонент полностью блокируется.

3. Техническая логика
3.1. Интеграция с Telegram
Использование скрипта https://telegram.org/js/telegram-widget.js.

После успешного входа — сохранение JWT-токена в localStorage или Cookies и перенаправление на главную страницу.

3.2. Работа с таймером (Экзамен)
При запуске варианта фронтенд получает end_time от сервера.

Визуальный таймер обратного отсчета в шапке.

При истечении времени — блокировка полей ввода и автоматическая отправка всех заполненных ответов.

3.3. Кэширование данных
Использование React Query (TanStack Query) для синхронизации с базой данных. Это позволит мгновенно отображать прогресс без лишних перезагрузок страницы.

4. Сценарии пользователя (User Stories)
Авторизация: Пользователь заходит на сайт -> Нажимает «Войти через TG» -> Получает доступ к списку задач.

Обучение: Выбирает «Разбор» -> Видит задачу -> Нажимает на ИИ-помощника -> Получает текстовое объяснение шагов.

Решение: Вводит число в таблицу -> Нажимает «Проверить» -> Видит конфетти (если верно) или красную подсветку (если ошибка).

Контроль: Переходит в раздел «Варианты» -> Подтверждает запуск таймера -> Решает 10 задач без возможности вызвать ИИ.

5. Дизайн-система (UI Kit)
Цвета: Спокойная палитра (синий/индиго для навигации, мягкий зеленый для успеха).

Шрифты: Читабельные без засечек (Inter, Roboto) для комфортного чтения условий задач.

Загрузка: Скелетоны (Skeletons) вместо спиннеров при загрузке тяжелых HTML-задач.
